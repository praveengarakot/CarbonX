# Project Concept

**CarbonX — Decentralized Carbon Credit Marketplace for SMEs**

### Problem

Large corporations can participate in carbon credit markets, but:

* Small businesses struggle to access them.
* Verification is expensive.
* Trading is fragmented.
* Carbon credit ownership is often opaque.

### Solution

A Stellar/Soroban-powered marketplace where:

* Verifiers certify carbon reductions.
* Carbon credits are tokenized on-chain.
* Businesses can buy/sell credits.
* Settlement happens automatically.

# Contract 1: Verification Contract

Purpose:

Verify that carbon reduction actually occurred.

Example:

```json
{
  "project": "Solar Plant Kolkata",
  "reduction": 120,
  "unit": "tCO2e"
}
```

Verifier approves.

Contract emits:

```rust
CarbonVerified
```

---

# Contract 2: Carbon Registry

Stores all carbon credits.

Example:

```json
{
  "credit_id": 102,
  "owner": "businessA",
  "amount": 120
}
```

When verification succeeds:

```text
Verification Contract
        ↓
Registry Contract
```

creates new credits automatically.

This is inter-contract communication.

---

# Contract 3: Marketplace

Businesses can:

* List credits
* Buy credits
* Cancel listings

Example:

```json
{
  "credit_id": 102,
  "price": 50 XLM
}
```

Events:

```rust
CreditListed
CreditPurchased
ListingCancelled
```

---

# Contract 4: Settlement Contract

Handles payment.

Flow:

```text
Buyer
   ↓
Settlement Contract
   ↓
Seller
```

Funds released only after ownership transfer.

Events:

```rust
PaymentLocked
PaymentReleased
```

---

# Event Streaming (Important)

Judges specifically want this.

Emit events for:

```rust
CarbonVerified
CreditMinted
CreditListed
CreditPurchased
OwnershipTransferred
PaymentReleased
```

Frontend listens in real time.

Show a live activity feed:

```text
✓ Credit #245 minted
✓ Credit #245 listed
✓ Credit #245 purchased
✓ Ownership transferred
```

This looks very impressive in demos.
